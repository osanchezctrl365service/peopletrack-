2 Crear el repositorio en GitHub

2.1 Andá a github.com → botón verde "New" → nombre: peopletrack → Create repository
2.2 En tu PC, abrí una terminal (cmd o PowerShell) y ejecutá:
mkdir peopletrack
cd peopletrack
git init
git remote add origin https://github.com/TU_USUARIO/peopletrack.git
2.3 Copiá los archivos del ZIP que te entregué dentro de la carpeta peopletrack. La estructura debe quedar así:
peopletrack/
  ├── frontend/
  │   ├── index.html           ← la app visual
  │   ├── api-client.js        ← cliente de API
  │   └── staticwebapp.config.json
  ├── api/
  │   ├── index.js             ← rutas del backend
  │   ├── db.js                ← conexión SQL
  │   ├── auth.js              ← validación Azure AD
  │   ├── package.json
  │   ├── host.json
  │   └── local.settings.json  ← NO subir a GitHub
  └── .github/
      └── workflows/
          └── deploy.yml
2.4 Agregá un archivo .gitignore en la raíz:
api/local.settings.json
api/node_modules/
*.env
2.5 Subí el código:
git add .
git commit -m "Initial PeopleTrack deploy"
git branch -M main
git push -u origin main

3 Crear la Azure Static Web App

3.1 Andá al portal.azure.com → buscá "Static Web Apps" en el buscador → + Create
3.2 Completá el formulario:
  • Subscription: tu suscripción
  • Resource Group: creá uno nuevo: rg-peopletrack
  • Name: peopletrack-app
  • Plan type: Free
  • Region: East US 2 (o la más cercana)
  • Source: GitHub → autorizá → seleccioná tu repo peopletrack → branch: main
  • App location: /frontend
  • Api location: /api
  • Output location: dejar vacío
3.3 Click en Review + Create → Create. Azure va a crear la app y agregar automáticamente un secret al repo de GitHub.
El primer despliegue tarda 3–5 minutos. Podés verlo en GitHub → Actions

5 Registrar la App en Azure AD (para SSO)

5.1 Portal Azure → Azure Active Directory → App registrations → + New registration
5.2 Completá:
  • Name: PeopleTrack
  • Supported account types: Accounts in this organizational directory only
  • Redirect URI: Single-page application → https://peopletrack-app.azurestaticapps.net
5.3 Una vez creada, copiá:
  • Application (client) ID → es tu AZURE_CLIENT_ID
  • Directory (tenant) ID → es tu AZURE_TENANT_ID
5.4 Andá a Certificates & secrets → + New client secret → copiá el valor → es tu AZURE_CLIENT_SECRET
5.5 Andá a API permissions → + Add a permission → Microsoft Graph → Delegated: openid, profile, email, User.Read → Grant admin consent
Actualizá AZURE_TENANT_ID, AZURE_CLIENT_ID y AZURE_CLIENT_SECRET en las Application Settings del paso 4

6 Habilitar MSAL en el frontend

6.1 Abrí frontend/index.html y agregá dentro del <head>:
<script src="https://alcdn.msauth.net/browser/2.38.3/js/msal-browser.min.js"></script>
<script src="api-client.js"></script>
6.2 En el <script> de la app, reemplazá la función doLogin() con:
async function doLogin() {
  API.initMsal(
    'TU_TENANT_ID',    // ← reemplazar
    'TU_CLIENT_ID',    // ← reemplazar
    window.location.origin
  );
  const account = await API.loginWithAzure();
  if (account) {
    document.getElementById('ssoScreen').style.display = 'none';
    const me = await API.getMe();
    if (me.ok) renderApp(me.data);
  }
}

async function renderApp(user) {
  // Cargar dashboard con datos reales
  document.querySelector('.user-info .name').textContent = user.FullName;
  document.querySelector('.user-info .role').textContent = user.RoleName + ' · ' + user.AreaName;
  document.querySelector('.user-avatar').textContent =
    user.FullName.split(' ').map(n => n[0]).join('').slice(0,2).toUpperCase();
  await loadDashboard();
}

async function loadDashboard() {
  const res = await API.getDashboard();
  if (!res.ok) return;
  // Los datos de res.data.team y res.data.kpi ya están listos
  // El renderTeamProgress() existente puede usarlos directamente
  renderTeamProgress(res.data.team);
  drawDonut(res.data.kpi);
}

7 Verificar el despliegue

7.1 Abrí GitHub → tu repo → pestaña Actions → verificá que el workflow "Deploy PeopleTrack" esté en verde ✅
7.2 Andá a tu URL: https://peopletrack-app.azurestaticapps.net
7.3 Probá el login con Azure AD con tu cuenta corporativa
7.4 Probá la API directamente:
https://peopletrack-app.azurestaticapps.net/api/periods
Si ves un JSON con los períodos fiscales, la API está funcionando correctamente

8 Agregar el primer usuario admin

8.1 Conectate a PeopleTrackDB con SSMS y ejecutá (reemplazando con tus datos):

VALUES (
  'tu-email@empresa.com',

Tu Azure Object ID lo encontrás en portal.azure.com → Azure Active Directory → Users → tu usuario → Object ID (campo en la página de perfil)
